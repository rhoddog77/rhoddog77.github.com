OVERVIEW


Like I said before, I creted this application as part of a team for my Android Application development class. As you can see, we provided screenshots of the app to the right because it is currently under development and not on the market.

The purpose of this app is to allow Musicians, Businesses, Restaurants, and other event holders to post the details of their event or gig for people to see. Users must register if they want to be able to upload a post, however, they can view posts without logging in.

PROCESS


You can see all of our commits on our Github repo.

We decided to use Git as our version control software throughout this project. It allowed us to work and upload changes to the project without fear of losing or breaking anything. We were also able to collaborate and see each other's changes as in the image above.

As far as some of the back end parts of the app goes we implemented Android's Holo design and layout. We created a custom list view for the main menu and incorporated Android's version of a session that allowed users to remain logged in when the closed the app. I personally built the back end server side PHP script which accessed our database and returned JSON objects to be parsed by our app. To prevent the app from freezing up when we accessed the database, we implemented an asynctask that runs on a seperate thread from the main app.


WHATS NEXT?


If you can't tell from the screen shots, this app has a bit of work to be done before it is fully optimized for use. We have basically an alpha version of the app running right now. This upcoming summer my partner and I are planning on adding some extra features, updating the design, and fixing any bugs we currently have. We want to provide the best user experience and design alongside a feature rich project to best serve our customers.
